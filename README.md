# AI Agent（龙虾）

## 「1人 → 3人团队 → 5人团队 → 小公司AI系统（龙虾）」架构演进


## 一人阶段（个人系统）
**→ 单机 AI 操作系统雏形**
### 目标
 <br> 先把 AI 系统 跑通 + 稳定运行，不追求复杂架构 
  
### 架构

```
你自己的主机（RTX4070 / 32GB）
├─ OpenClaw
├─ MiniMax / OpenAI API
├─ Ollama（可选）
├─ 本地代码 / 日志 / 脚本
└─ 自动化任务
```

### 能力范围

* 日常对话 + 任务拆解
* 代码生成 / Shell / Python 执行
* 本地模型实验（Ollama）
* 一台机器控制另一台机器（SSH / API）

### 核心稳定点
1. Gateway 生命周期管理（启动 / 停止 / 重启）
2. 模型切换能力（云 / 本地）
3. 自动化执行闭环（输入 → 执行 → 输出）



##　三人阶段（团队 AI 中枢）

**→AI 中枢 + 多终端执行网络**

###　目标
 <br> 从个人工具 → 团队共享 AI 系统


 ###　架构
 ```
 主控机（RTX4070 / 4080）
├─ OpenClaw Gateway
├─ API模型（MiniMax / OpenAI）
├─ 本地模型（Ollama）
└─ 任务路由

成员A电脑
└─ 开发 / 测试 / 执行

成员B电脑
└─ 日志分析 / 脚本

成员C电脑
└─ 文档 / 运维 / 轻量任务
```
###　执行流程

```
OpenClaw（主控）
   ↓
任务拆解
   ↓
SSH / API 分发
   ↓
成员机器执行
   ↓
结果回传
```


###　分工

* 主控机：统一管理 AI / 模型 / 技能
* 成员机器：执行节点（轻量）
* 云 API：复杂推理
* 本地模型：低成本任务


### 核心升级点
* 技能库共享（SKILL.md）
* 节点执行
* 简单调度机制

## 五人阶段（分布式 AI 系统）
**→小型分布式 AI 系统**

### 目标
<br> 从共享工具 → 分布式 AI 系统
### 架构

#### 主控层（Control Layer）
```
AI 主控机(4070/4080)
├─ OpenClaw Gateway
├─ 本地模型（Ollama）
├─ 技能库
├─ 任务调度
└─ 权限控制

```

#### 执行层（Execution Layer）
```
5台执行机(mac)
├─ 脚本执行
├─ 自动化测试
├─ 数据处理
└─ 本地模型推理
```
#### 云层（Cloud Layer）
```
MiniMax / OpenAI / 云GPU
├─ 复杂推理
├─ 高峰负载
└─ 大模型能力
```

#### 执行流
```
用户 / Agent
   ↓
OpenClaw（主控）
   ↓
任务拆解 + 调度
   ↓
执行层
   ↓
云模型（必要时）
   ↓
结果返回
```

核心升级点

* 调度系统（Scheduler）
* 权限控制
* 并行执行
* 负载分配（本地 vs 云）



### 小公司 AI 系统（最终形态）
**→企业级 AI 操作系统（AI OS）**
#### 目标
<br>构建企业级 AI 基础设施

#### 架构
```
┌──────────────────────┐
│     云模型层（可选）   │
│ GPT / Claude / MiniMax│
└──────────┬───────────┘
           │
           ▼
┌────────────────────────────┐
│ 主控层（AI 控制中枢）        │
│ OpenClaw Gateway           │
│ - 任务拆解                 │
│ - Agent编排                │
│ - 模型路由                 │
└──────────┬───────────┘
           │
           ▼

┌──────────────────────┐   ┌──────────────────────┐
│ 推理节点 A（4090）    │   │ 推理节点 B（4070）    │
│ 重任务 / 大模型       │   │ 日常任务 / 并发       │
└──────────────────────┘   └──────────────────────┘

           │
           ▼

┌────────────────────────────┐
│ 执行节点（Worker）          │
│ Shell / Python / CI/CD      │
│ 数据处理                    │
│ 自动化任务                  │
└────────────────────────────┘
```
### 核心能力
* Multi-Agent 编排
* 模型路由（云 + 本地）
* 自动化运维
* AI 辅助开发
* 企业知识库


### 设计原则
1. 控制层 ≠ 执行层
2. 云 + 本地混合
3. 可调度 / 可扩展
4. 无单点依赖

# AIエージェント(Lobster)

## 「1人→3人チーム→5人チーム→小規模企業のAIシステム(龍蝦)」アーキテクチャの進化


## 1人フェーズ（個人システム）
**→ 単一マシン AI オペレーティングシステムの原型**
### 目標
 <br> まず AI システムを動作させ、安定稼働させること。複雑なアーキテクチャは追求しない
 
### アーキテクチャ

```
自分のホスト（RTX4070 / 32GB）
├─ OpenClaw
├─ MiniMax / OpenAI API
├─ Ollama（任意）
├─ ローカルコード / ログ / スクリプト
└─ 自動化タスク
```

### 対応範囲

* 日常会話 + タスク分解
* コード生成 / Shell / Python 実行
* ローカルモデル実験（Ollama）
* 1台のマシンで別のマシンを制御（SSH / API）


### 安定稼働
1. Gateway ライフサイクル管理（起動 / 停止 / 再起動）
2. モデル切替機能（クラウド / ローカル）
3. 自動化実行のクローズドループ（入力 → 実行 → 出力）



##　3人フェーズ（チームAIハブ）

**→AIハブ ＋ マルチ端末実行ネットワーク**

###　目標
 <br> 個人ツールから、チームで共有するAIシステムへ


 ###　アーキテクチャ
 ```
　メイン制御機（RTX4070 / 4080）
├─ OpenClaw Gateway
├─ APIモデル（MiniMax / OpenAI）
├─ ローカルモデル（Ollama）
└─ タスクルーティング

メンバーAのPC
└─ 開発 / テスト / 実行

メンバーBのPC
└─ ログ分析 / スクリプト

メンバーCのPC
└─ ドキュメント / 運用 / 軽量タスク
```
###　実行フロー

```
OpenClaw（メイン制御）
   ↓
タスク分解
   ↓
SSH / API 配信
   ↓
メンバーマシンで実行
   ↓
結果を返送
```


###　役割分担

* メイン制御機：AI / モデル / スキルの統合管理
* メンバーマシン：実行ノード（軽量）
* クラウド API：高度な推論
* ローカルモデル：低コストタスク


### 核主なアップグレード点
* スキルライブラリ共有（SKILL.md）
* ノード実行
* シンプルなスケジューリング機構

## 5人フェーズ（分散型AIシステム）
**→小規模分散型 AI システム**

### 目標
<br> 共有ツールから、分散型AIシステムへ
### アーキテクチャ

#### 制御層 (Control Layer)
```
AI メイン制御機(4070/4080)
├─ OpenClaw Gateway
├─ ローカルモデル（Ollama）
├─ スキルライブラリ
├─ タスクスケジューリング
└─ 権限管理


```

#### 実行層 (Execution Layer)
```
5台の実行機(mac)
├─ スクリプト実行
├─ 自動テスト
├─ データ処理
└─ ローカルモデル推論
```
#### クラウド層 (Cloud Layer)
```
MiniMax / OpenAI / クラウドGPU
├─ 高度な推論
├─ ピーク負荷対応
└─ 大規模モデル機能
```

#### 実行フロー
```
ユーザー / Agent
   ↓
OpenClaw（メイン制御）
   ↓
タスク分解 + スケジューリング
   ↓
実行層
   ↓
クラウドモデル（必要に応じて）
   ↓
結果返却
```

核心的なアップグレード点

* スケジューリングシステム（Scheduler）
* 権限管理
* 並列実行
* 負荷分散（ローカル vs クラウド）



### 小規模企業のAIシステム（最終形態）
**→企業級 AI オペレーティングシステム（AI OS）**
#### 目標
<br>企業級AIインフラストラクチャの構築

#### アーキテクチャ
```
┌──────────────────────┐
│     クラウドモデル層（任意）   │
│ GPT / Claude / MiniMax│
└──────────┬───────────┘
           │
           ▼
┌────────────────────────────┐
│ メイン制御層（AI コントロール中枢）        │
│ OpenClaw Gateway           │
│ - タスク分解                 │
│ - Agent編成                │
│ - モデルルーティング                 │
└──────────┬───────────┘
           │
           ▼

┌──────────────────────┐   ┌──────────────────────┐
│ 推論ノード A（4090）    │   │ 推論ノード B（4070）    │
│ 重負荷タスク / 大規模モデル       │   │ 日常タスク / 並列処理       │
└──────────────────────┘   └──────────────────────┘

           │
           ▼

┌────────────────────────────┐
│ 実行ノード（Worker）          │
│ Shell / Python / CI/CD      │
│ データ処理                    │
│ 自動化タスク                  │
└────────────────────────────┘
```
### コア機能
* Multi-Agent 編成
* モデルルーティング（クラウド + ローカル）
* 自動運用
* AI 支援開発
* 企業ナレッジベース


### 设计原则
1. 制御層 ≠ 実行層
2. クラウド + ローカルのハイブリッド
3. スケジューラブル / 拡張可能
4. 単一障害点なし
